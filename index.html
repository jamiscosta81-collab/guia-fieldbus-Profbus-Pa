Conexão do DM-AM ao segmento
Corrente mínima via USB: 150 mA. Verifique o manual do notebook. Se insuficiente, use um hub USB com fonte externa.
Tela real capturada em campo
Tela real do PACTware — árvore de projeto à esquerda: PC HOST → FDS → DM-AM → FDSCommCI · tipo de barramento: PROFIBUS PA · qualidade atual do segmento visível no topo
  1. Conectar o cabo fieldbus ao segmento a ser analisado (pontos de teste ou cabo direto)
  2. Encaixar o conector do cabo fieldbus na entrada do DM-AM (pólos + / − / Shield)
  3. Conectar o DM-AM ao notebook via cabo USB 2.0 fornecido no kit
  4. Aguardar o LED PWR acender vermelho — driver instalado automaticamente pelo Windows (Plug & Play)
  5. Verificar o LED COM/ERR — verde = segmento detectado · piscando = alarme presente
  6. Abrir o PACTware e garantir que o FDS está rodando (ícone na barra de tarefas)
  7. No projeto, clicar com botão direito no DM-AM → Connect
Uso do Test Plug (ponteiras de teste)
As ponteiras de teste encaixam nos pontos de teste dos conectores Pepperl+Fuchs — não é padrão industrial mas facilita muito a conexão em campo sem desligar o barramento.
  1. Separar as 3 ponteiras modulares do conjunto
  2. Retirar a parte metálica de cada ponteira
  3. Passar um fio por cada ponteira e soldar na parte metálica
  4. Rosquear a parte metálica de volta na ponteira
  5. Encaixar as 3 ponteiras juntas na ordem: + / − / S (Shield)
  6. Inserir o conjunto nos pontos de teste do conector fieldbus
Montagem em trilho DIN (data-logger)
A montagem com clip no trilho DIN é permitida apenas para medições temporárias. Não usar onde houver vibração.
  1. Encaixar o mounting clip no trilho DIN até travar (Click!)
  2. Pressionar o DM-AM no mounting clip até travar (Click!)
  3. Verificar se está firme antes de conectar os cabos

Quando usar o osciloscópio
Use o osciloscópio para análise profunda do sinal — quando o Segment Monitoring mostra algo anormal e você precisa ver exatamente o que está acontecendo no barramento.
Exemplo de captura — barramento FF/PA
Forma de onda típica de um segmento FF H1 saudável — sinal limpo, amplitude dentro do especificado
Como abrir
  1. Clicar com botão direito no DM-AM no projeto
  2. Selecionar Additional functions → Fieldbus Oscilloscope
  3. Na janela do osciloscópio, configurar os parâmetros desejados
  4. Clicar em Start para iniciar a captura

🔴 Caça ao problema intermitente — "pisca e some"
Sintoma clássico: alarme aparece no Diagnostics mas quando você abre o PACTware o segmento está normal. O problema acontece de forma esporádica — pode demorar horas ou dias para se repetir. O osciloscópio com trigger é a única ferramenta que consegue capturar o evento exato.
Fluxo de diagnóstico — do alarme no Diagnostics até a captura do evento intermitente no osciloscópio
Passo a passo — capturar evento intermitente
Antes de começar: deixe o Diagnostics aberto em paralelo com o osciloscópio. Quando o trigger disparar, anote imediatamente o horário para cruzar com o histórico de alarmes.
① Abrir o osciloscópio
  1. Com o DM-AM conectado ao segmento e ao notebook, abrir o PACTware
  2. Clicar com botão direito no DM-AM → Additional functions → Fieldbus Oscilloscope
  3. A janela do osciloscópio abre mostrando o sinal em tempo real — ainda sem trigger configurado
② Configurar o trigger para o evento intermitente
  1. No campo Trigger Event, selecionar o evento que você quer capturar (ver tabela acima)
  2. Se o problema é device sumindo da rede: selecionar "Missing Token Usage" (FF) ou "Missing Response" (PA) e digitar o endereço do device suspeito no campo Trigger Address
  3. Se o problema é ruído/spike: selecionar "Signal Level" e ajustar o Trigger Level para +0.45 V (captura quando o sinal ultrapassa o limite)
  4. Se o problema é CRC / Framing Error: selecionar "CRC Error" diretamente — o osciloscópio vai aguardar qualquer frame com erro
  5. Ajustar o Trigger Timeout para Infinite — o osciloscópio ficará aguardando até o evento aparecer, sem parar sozinho
  6. Aumentar o Pretrigger Time ao máximo — assim você vê o que aconteceu no barramento antes do evento, não só depois
  7. Aumentar o Recording Length se quiser mais contexto — valores maiores reduzem o sample rate mas capturam mais tempo
③ Iniciar a espera e capturar
  1. Clicar em Start — o status muda para Waiting for trigger...
  2. Deixar o notebook conectado e não fechar o PACTware — o osciloscópio fica aguardando em background
  3. Quando o evento ocorrer, o trigger dispara automaticamente e a captura é exibida na tela
  4. O status muda para Triggered! — a forma de onda capturada aparece congelada
Se o problema ocorre em horário específico (ex: turno da tarde, período de produção intensa), configure o trigger antes desse horário e aguarde. O osciloscópio não consome recursos significativos enquanto espera.
④ Analisar e salvar a captura
  1. Com a captura exibida, observar a forma de onda — procurar o ponto exato onde o sinal saiu do padrão
  2. Usar o cursor (arraste na tela) para medir a amplitude do spike ou a duração da anomalia
  3. Anotar o endereço do device que transmitia no momento do evento (visível no frame capturado)
  4. Clicar em Save As para salvar a imagem da captura como arquivo — escolher um nome com data/hora (ex: Seg01_CRCError_15jan_14h32.osc)
  5. Cruzar o horário da captura com o histórico no Diagnostics para confirmar a correlação do alarme com o evento físico
  6. Se quiser repetir a captura, clicar em Restart — o trigger volta para o modo de espera
⑤ Identificar o causador
  1. Se o evento é sempre precedido pelo mesmo endereço de device → device causador identificado — substituir ou inspecionar o spur daquele device
  2. Se o spike aparece sem transmissão de nenhum device → ruído externo — inspecionar aterramento, blindagem e proximidade com cabos de potência
  3. Se o evento ocorre em horário de liga/desliga de equipamentos → interferência elétrica — confirmar com osciloscópio de potência no quadro elétrico
  4. Se múltiplos devices somem ao mesmo tempo → problema no PSU ou no trunk — verificar tensão no segmento com multímetro no momento do evento
  5. Enviar o arquivo .osc salvo ao suporte Pepperl+Fuchs se não conseguir identificar a causa — eles conseguem analisar remotamente
Se os CRC Errors ocorrem sempre associados ao mesmo endereço de device: desconecte esse device do spur temporariamente e monitore o segmento. Se os erros desaparecem, o device está com o transmissor degradado e precisa ser substituído.

Alarmes de segmento — causa e ação
📶
Signal Level High
Falta um terminador — apenas 1 no lugar de 2 · cabo rompido ou desconectado
→ Verificar terminadores (1 em cada extremidade do trunk) · verificar integridade do cabo · usar osciloscópio
📉
Signal Level Low
Mais de 2 terminadores · umidade/sujeira no cabo (carga polo-a-polo) · posição de device alterada
→ Confirmar exatamente 2 terminadores · verificar cabeamento · confirmar posição dos devices no trunk
〰️
Noise Level High
Blindagem ou aterramento incorreto · cabo AC próximo ao barramento · equalização de potencial ausente
→ Verificar aterramento e blindagem · afastar cabos de alta corrente · confirmar tipo correto de cabo · usar osciloscópio
⏱️
Jitter Level High
Terminação errada · cabo inadequado · trunk ou spurs muito longos · dois segmentos em paralelo
→ Verificar terminação e tipo de cabo · revisar topologia · identificar device causador pelo osciloscópio e substituir se necessário
⚖️
DC Unbalance Alarm
Cabeamento incorreto (polo negativo ligado ao shield) · dano no cabo · device com conexão assimétrica
→ Verificar cabeamento do segmento · desconectar devices um a um para identificar o causador · inspecionar caixas de junção por umidade
⚡
Segment Voltage Alarm
PSU com problema · tipo incorreto de PSU · bulk supply com falha
→ Verificar tensão no terminal do segmento com multímetro · conferir tipo e funcionamento do PSU
🔋
Segment Current High
Carga extra no barramento · curto em cabo longo · falha de device · ingresso de água em caixa de junção
→ Verificar se há carga adicional no barramento · inspecionar caixas de junção · isolar devices um a um

📡
Device Signal Level High / Low
Posição do device no trunk · problema de transmissão do device específico
→ Verificar endereço do device causador no histórico · checar posição no trunk · se necessário substituir o device

  2. Clicar em Export (ícone de salvar)
  3. Escolher diretório, nome do arquivo e extensão .txt

O que é um Snapshot
O Snapshot captura todos os valores atuais do segmento — tensão, corrente, noise, jitter, signal level de cada device — e gera um relatório de Physical Layer que pode ser salvo, impresso ou enviado como PDF ou Excel.
Como criar um Snapshot
  1. Abrir o Segment Monitoring (botão direito no DM-AM → Additional functions → Segment Monitoring)
  2. Aguardar os valores estabilizarem em tempo real
  3. Clicar em "Create Snapshot" no topo da janela
  4. Aguardar a barra de progresso completar (captura todos os devices)
  5. No campo Description, dar um nome ao relatório (ex: Seg01_Planta_A_Jan2025)
  6. Clicar em Save — abre automaticamente o Snapshot Explorer
Como exportar o relatório (Snapshot Explorer)
  1. Abrir o Snapshot Explorer: botão direito no DM-AM → Additional functions → Snapshot Explorer
  2. Selecionar o relatório desejado na lista à esquerda
  3. Visualizar o preview à direita

Como ler o relatório offline (sem PACTware)
Exporte o relatório como PDF — ele pode ser aberto em qualquer dispositivo (celular, tablet, notebook) sem precisar do PACTware instalado. Ideal para consulta no campo ou envio para outros técnicos.
  1. Exportar o Snapshot como PDF conforme descrito acima
  2. Salvar o PDF em uma pasta de fácil acesso (ex: Desktop ou OneDrive)
  3. Para acessar no celular: enviar por WhatsApp, e-mail ou salvar no Google Drive
  4. O PDF contém todos os valores medidos, classificação de qualidade e data/hora da captura

Problema intermitente — não aparece na hora (referência rápida)
👻
Alarme aparece mas segmento parece normal quando você abre o PACTware
O problema já passou — você está vendo o "fantasma" no histórico do Diagnostics. O evento real foi rápido demais para pegar em tempo real.
→ Ir para a aba Oscilloscope · configurar trigger adequado · deixar em modo "Waiting for trigger... Infinite" · aguardar o evento se repetir · ver passo a passo completo na aba Oscilloscope
⏰
Evento ocorre em horário específico (ex: início de turno)
Provavelmente interferência elétrica de motor, inversor de frequência ou resistência de aquecimento que liga nesse horário
→ Configurar o trigger antes do horário suspeito · usar trigger "Signal Level" com nível +0.45 V · se confirmar interferência, verificar aterramento e blindagem do cabo no trecho próximo ao equipamento
🔁
Device some e volta periodicamente
Conexão mecânica fraca no spur · umidade no terminal · device com problema de transmissor
→ Trigger: "Missing Response" (PA) ou "Missing Token Usage" (FF) com o endereço do device suspeito · quando capturar, verificar se o sinal do device estava ausente ou distorcido antes do desaparecimento
Sequência de checkout em campo
  1. Cabo inicial (sem PSU/devices): verificar continuidade, terminadores, capacitância e aterramento do shield com multímetro
  2. Checkout de segmento (energizado + DM-AM): verificar tensão, corrente e DC unbalance no Segment Monitoring
  3. Checkout de rede (host conectado): verificar signal level, noise, jitter e polarity no Segment Monitoring · usar osciloscópio para CRC e Framing errors
  4. Checkout por device (slaves conectados): verificar signal level e jitter individual de cada device · identificar causadores de alarmes

Physical Layer — Camada Física
Signal Level Physical Layer 200–1200 mV
Amplitude do sinal digital no barramento, medida em milivolts (mV). Representa a diferença de tensão entre os pólos + e − durante a transmissão de um bit. É o parâmetro mais crítico para garantir que todos os devices consigam ler os dados corretamente.
Alto (>1200 mV): falta um terminador ou cabo rompido. Baixo (<200 mV): terminadores em excesso, umidade no cabo ou distância excessiva.
Noise Physical Layer <100 mV
Tensão parasita presente no barramento quando nenhum device está transmitindo. Originado por campos eletromagnéticos externos (motores, inversores de frequência, fontes chaveadas) que induzem tensão no cabo. Valores altos de noise aumentam a taxa de erros de comunicação.
Acima de 100 mV: verificar aterramento e blindagem do cabo, afastar de cabos de potência, confirmar equalização de potencial.
Jitter Physical Layer <2,4 µs
Variação no tempo de chegada dos bits em relação ao tempo esperado, medida em microssegundos (µs). Em um barramento ideal cada bit dura exatamente 32 µs (FF H1 a 31,25 kbit/s). O jitter é a diferença entre o tempo real e esse ideal. Valores altos indicam problemas na terminação ou distorção do sinal ao longo do cabo.
Acima de 3,2 µs (Out of Spec IEC): verificar terminadores, tipo de cabo e comprimento do trunk/spurs.
DC Unbalance Physical Layer Alarme crítico
Desequilíbrio DC entre os dois pólos do barramento (+ e −) em relação ao shield/terra. Em um barramento corretamente instalado, os dois pólos devem estar igualmente isolados do shield. Um valor de DC Unbalance elevado indica que um dos pólos está com resistência menor para o terra — geralmente por umidade, sujeira ou curto parcial no cabo ou em uma caixa de junção.
Como isolar: desconectar os devices um a um. Quando o DC Unbalance voltar a zero, o último device desconectado ou seu spur é o causador.
Terminador (Terminator) Physical Layer Sempre 2 no trunk
Resistor de 100 Ω com capacitor em série (RC) instalado nas duas extremidades do trunk do segmento. Sua função é absorver o sinal no final do cabo, evitando reflexões que causariam interferência com os frames seguintes. Um segmento FF H1 ou PROFIBUS PA deve ter exatamente 2 terminadores — um em cada ponta do trunk.
1 terminador: Signal Level alto. 3 ou mais: Signal Level baixo. 0 terminadores: comunicação impossível.
Trunk Physical Layer
Cabo principal do segmento — a "espinha dorsal" que vai da sala de controle até a área de campo, passando pelas caixas de junção. Os terminadores ficam nas duas extremidades do trunk. O comprimento máximo do trunk depende do tipo de cabo utilizado (Tipo A: até 1900 m para FF H1).
Spur Physical Layer Máx. 120 m
Cabo de derivação que sai do trunk (via caixa de junção) até o field device. É uma ramificação do trunk — não tem terminador. O comprimento máximo de cada spur é 120 m para FF H1. Spurs muito longos aumentam o jitter e podem causar reflexões de sinal.
Comunicação — Protocolo
CRC Error Comunicação Erro crítico
Cyclic Redundancy Check — cada frame transmitido no barramento contém um valor de verificação calculado a partir dos dados. O receptor recalcula esse valor e compara. Se os valores não batem, é um CRC Error: o frame foi corrompido em trânsito (por ruído, spike ou distorção do sinal). Um ou dois erros esporádicos podem ser tolerados; erros frequentes indicam problema sério no barramento.
Para capturar: osciloscópio com trigger "CRC Error" + Timeout Infinite. O frame corrompido aparece na tela exatamente no momento em que ocorreu.
Framing Error Comunicação Erro crítico
Erro na estrutura do frame — os bits de sincronismo (start/stop) não foram recebidos corretamente. Enquanto o CRC Error indica dado corrompido, o Framing Error indica que o receptor nem conseguiu sincronizar com o transmissor. Causado por distorção severa do sinal, jitter muito alto ou device com transmissor com defeito.
Token (Pass Token) Comunicação FF H1
No FOUNDATION Fieldbus, o LAS (Link Active Scheduler — o "mestre" do barramento) passa um "token" para cada device quando é a vez daquele device transmitir. O token é uma mensagem de permissão. Se um device não usa o token no tempo esperado, o LAS marca como "Missing Token Usage" — indica que o device não respondeu à sua vez de transmitir.
Missing Response Comunicação PROFIBUS PA
No PROFIBUS PA, o master DP envia uma requisição a um slave e aguarda a resposta dentro de um tempo máximo (slot time). Se o slave não responder nesse tempo, é um Missing Response — o device foi declarado ausente. Causas: device desligado, conexão física ruim, device com falha, ou interferência no spur.
Polarity Comunicação
Indica se os pólos + e − do spur de um device estão conectados corretamente. FF H1 e PROFIBUS PA são tolerantes à inversão de polaridade (o device funciona mesmo com + e − invertidos), mas o DM-AM detecta e indica quando há inversão — útil para encontrar erros de fiação durante o comissionamento.
Equipamentos e Software
PSU Equipamento
Power Supply Unit — fonte de alimentação do segmento. Injeta tensão DC (tipicamente 24–32 V) no barramento fieldbus e também limita a corrente máxima disponível. Nos sistemas Pepperl+Fuchs FieldConnex, o PSU tem impedância de acoplamento interna que isola o sinal do barramento do barulho da fonte de alimentação.
FDS Software
FieldConnex Diagnostic Server — serviço Windows instalado junto com o software do DM-AM. Fica rodando em background (ícone na barra de tarefas do Windows) e gerencia a comunicação USB entre o DM-AM e o PACTware. Sem o FDS rodando, o DM-AM não aparece no PACTware.
O ícone do FDS deve estar verde na barra de tarefas. Se estiver vermelho ou ausente, reiniciar o serviço pelo Gerenciador de Tarefas do Windows.
PACTware Software
Software FDT (Field Device Tool) frame — plataforma aberta que hospeda os DTMs (Device Type Managers) de diferentes fabricantes. É a interface principal para configurar, diagnosticar e monitorar o DM-AM e os field devices do segmento. Versão mínima recomendada: PACTware 5.0.
DTM Software
Device Type Manager — equivalente a um "driver" de um instrumento de campo para o PACTware. Cada fabricante disponibiliza o DTM do seu instrumento, que quando instalado no PACTware permite parametrizar, calibrar e diagnosticar aquele device específico. O DM-AM também tem seu próprio DTM.
FDT Padrão
Field Device Tool — padrão aberto que define a interface entre software de configuração (frame, como o PACTware) e os DTMs dos fabricantes. Garante interoperabilidade: o DTM de qualquer fabricante que siga o padrão FDT pode rodar dentro do PACTware ou de qualquer outro frame FDT.
Protocolos de Barramento
FOUNDATION Fieldbus H1 (FF H1) Protocolo 31,25 kbit/s
Protocolo de comunicação digital para instrumentação de processo, desenvolvido pela Fieldbus Foundation. Opera a 31,25 kbit/s, alimentação e comunicação no mesmo par de fios, tensão típica de 24–32 V no segmento. Suporta até 32 devices por segmento (na prática 16 para ter margem). Muito usado em indústrias de petróleo, gás e química.
PROFIBUS PA Protocolo 31,25 kbit/s
Process Automation — variante do PROFIBUS para instrumentação de processo. Também opera a 31,25 kbit/s com alimentação no mesmo cabo. Usa o mesmo Physical Layer do FF H1 (IEC 61158-2), por isso os parâmetros de diagnóstico (Signal Level, Noise, Jitter) são os mesmos. A diferença está na camada de aplicação e no protocolo de comunicação (master/slave vs. token-passing).
IEC 61158-2 Norma
Norma internacional que define o Physical Layer do FF H1 e PROFIBUS PA — incluindo os limites de Signal Level, Noise, Jitter e DC Unbalance. Quando o DM-AM indica "Out of Spec", significa que o valor medido ultrapassou o limite definido por essa norma. É a referência usada pelo Show Wizard no PACTware para classificar a qualidade do segmento.
LAS Protocolo FF H1
Link Active Scheduler — o "mestre" do segmento FOUNDATION Fieldbus. Controla quem transmite e quando, passando o token para cada device na ordem configurada. Normalmente é o host (sistema de controle) ou um device configurado como LAS. Se o LAS falhar, o segmento inteiro perde comunicação.
Diagnóstico e Ferramentas DM-AM
Snapshot Função DM-AM
Fotografia instantânea de todos os parâmetros do segmento no momento da captura — tensão, corrente, noise, jitter, signal level individual de cada device. Gera um relatório de Physical Layer que pode ser exportado como PDF, Excel ou enviado ao suporte técnico. Essencial para documentar o comissionamento e comparar medições ao longo do tempo.
Segment Monitoring Função DM-AM
Janela principal do DM-AM no PACTware. Exibe em tempo real todos os parâmetros do segmento com classificação de qualidade (Excellent / Good / Out of Spec). É o ponto de partida para qualquer diagnóstico — abrir aqui primeiro, depois usar o Oscilloscope ou Diagnostics para aprofundar.
Show Wizard Função DM-AM
Botão no Segment Monitoring que ativa as barras coloridas de qualidade ao lado de cada valor. Mostra visualmente se o parâmetro está dentro da faixa ideal (verde), aceitável (cinza) ou fora da especificação IEC (amarelo/vermelho). Facilita a leitura rápida em campo sem precisar memorizar os limites numéricos.
Data Logger Função DM-AM
Modo de operação autônoma do DM-AM — sem necessidade de notebook conectado. Alimentado pelo AC/DC adapter externo, o módulo registra os parâmetros do segmento em intervalos configuráveis (4 h, 8 h, 12 h, 24 h…) em sua memória interna. Ideal para monitoramento de longa duração. Para ler os dados, conectar depois via USB ao PACTware e usar History Data Export.
Trigger Oscilloscope
Condição que faz o osciloscópio do DM-AM iniciar a captura da forma de onda. Em vez de capturar continuamente (o que geraria dados demais), o osciloscópio fica em espera e só grava quando o evento definido ocorre — ex: um CRC Error, um sinal acima de 0,45 V ou um device que não respondeu. É a ferramenta essencial para capturar eventos intermitentes que não são visíveis em tempo real.
Pretrigger Time Oscilloscope
Quanto tempo antes do evento de trigger o osciloscópio inclui na captura. Por exemplo: com Pretrigger = 50%, a captura mostra o barramento 50% do tempo antes do evento e 50% depois. Aumentar o Pretrigger é essencial para ver o que estava acontecendo no barramento imediatamente antes do problema — o contexto que antecedeu o evento.
DMS Formato de arquivo
Formato de arquivo proprietário Pepperl+Fuchs para exportação de diagnóstico do DM-AM. Contém todos os dados do segmento em formato compactado. Ao abrir um chamado no suporte técnico da Pepperl+Fuchs, solicitar o envio do arquivo .DMS — eles conseguem visualizar todos os dados do segmento remotamente sem precisar ir ao campo.
